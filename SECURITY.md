# Security Policy for Gamatrain Governance

## Supported Versions

This section tells people which versions of Gamatrain Governance are currently supported with security updates.

| Version | Supported          |
| ------- | ----------------- |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:               |

---

## Reporting a Vulnerability

If you discover a security vulnerability in Gamatrain Governance:

1. **Do not exploit it.**  
   Exploiting vulnerabilities can harm users and may be illegal.

2. **Submit a report** through our contact page:  
   [https://gamatrain.com/contact-us](https://gamatrain.com/contact-us)

3. Include as much detail as possible:  
   - Description of the vulnerability.  
   - Steps to reproduce the issue.  
   - Any relevant screenshots, logs, or code snippets.  
   - Potential impact if exploited.  

**Safe Handling:**  
- Reports will be handled confidentially.  
- Do not post vulnerabilities publicly before they are fixed.  
- We aim to acknowledge receipt within 48 hours and provide updates on the report status.

---

## What We Expect From Reporters

- Ethical behavior and discretion.  
- Clear, accurate, and detailed reporting.  
- No social engineering or privacy violations.  

---

## Rewards and Recognition

We value contributions from the community. For eligible reports:  

- Public acknowledgment (if desired).  
- Rewards or bounties may be offered depending on severity.  

---


We really appreciate your help in keeping Gamatrain Governance safe!  
If you find something, please report it responsibly so we can fix it quickly and protect our community. Thank you for being part of our journey!
